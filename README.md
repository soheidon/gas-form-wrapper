# gas-form-wrapper（Google フォーム用カスタム回答 UI）

Google Workspace 組織内の Web アプリとして動かし、**既存の Google フォームの設問定義を読み取り、カスタム画面から回答を受け付けてフォームへ送信**します。設問の編集は引き続き Google フォーム側で行えます。

## GASのデプロイ

### 1. 公開までの流れ（イメージ）

1. **Googleフォーム**と、**回答やログを確認するための Google スプレッドシート**を、あらかじめ用意しておきます（中身の詳しい決まりは、組織の担当者から案内がある場合があります）。
2. このツールは **Google の「アプリ」として Web 上に公開**され、**ブラウザで開くための URL（リンク）** が発行されます。  
   - すでに **IT 担当や開発担当から URL を渡されている**場合は、この手順は不要で、そのリンクを開いてください。  
   - **ご自身で公開する場合**は、Google Apps Script の**プロジェクトをブラウザで開いたうえで**、メニューから **ウェブアプリとしてデプロイ**します。初めて開くと「権限の確認」などの案内が出ることが多いです。**操作の詳細は、この節の先頭のリンク**（とくに [webapp-deploy-for-beginners.md](docs/webapp-deploy-for-beginners.md)）を参照してください。
3. 次の **「初期設定画面」** で、フォームとスプレッドシートの URL を登録します。

### 2. 初期設定画面の開き方

| 状況 | どう開くか |
|------|------------|
| **初回**（管理者メールがまだ登録されていない） | Web アプリの **通常の URL** を開くだけでよい。アドレスに **`?page=setup` は不要**で、自動的に初期設定画面が表示される |
| **2 回目以降・設定の変更** | URL の末尾に **`?page=setup`** を付けて開く。**初回に設定を保存した Google アカウント**だけが開ける |

回答者に配るのは、初期設定が終わったあとに画面に表示される **回答用の URL**（`?page=setup` が付いていない通常のアドレス）です。

### 3. 初期設定画面に表示される内容

画面上の見出しと入力の意味は次のとおりです。

#### 初期設定の前に準備しておくもの

- **Googleフォーム**  
  質問項目や回答形式を含めて、あらかじめ完成させておいてください。

- **回答を確認するための Google スプレッドシート**  
  回答内容やログを確認するために使います。事前に作成しておいてください。

#### 初期設定

フォームとスプレッドシートの URL を貼り付けて **「設定を保存する」** を押すと、**回答者に送る用の URL** が表示されます。

#### 入力欄

- **GoogleフォームのURL（必須）**  
  使いたい Google フォームをブラウザで開き、アドレス欄の URL をそのまま貼り付けてください。

- **GoogleスプレッドシートのURL（必須）**  
  ログ確認に使う Google スプレッドシートを開き、その URL をそのまま貼り付けてください。

保存に成功すると、以降は同じ Web アプリの通常 URL で **回答画面**が開くようになります。
