# gas-form-wrapper（Google フォーム用カスタム回答 UI）

Google Workspace 組織内の Web アプリとして動かし、**既存の Google フォームの設問定義を読み取り、カスタム画面から回答を受け付けてフォームへ送信**します。設問の編集は引き続き Google フォーム側で行えます。

## GASのデプロイ

**Google Apps Script をウェブアプリとして公開する**手順の詳細は、次のドキュメントを参照してください。

- **[docs/webapp-deploy-for-beginners.md](docs/webapp-deploy-for-beginners.md)** — 編集画面からのデプロイ（一般向け・画面の流れ・つまずき・公式ヘルプへのリンク）
- **[ウェブアプリとしてデプロイする（Google 公式・日本語）](https://developers.google.com/apps-script/guides/web?hl=ja)** — Apps Script の公式ガイド


### 1. 公開までの流れ（イメージ）

1. **Googleフォーム**と、**回答やログを確認するための Google スプレッドシート**を、あらかじめ用意しておきます（中身の詳しい決まりは、組織の担当者から案内がある場合があります）。
2. このツールは **Google の「アプリ」として Web 上に公開**され、**ブラウザで開くための URL（リンク）** が発行されます。  
   - すでに **IT 担当や開発担当から URL を渡されている**場合は、この手順は不要で、そのリンクを開いてください。  
   - **ご自身で公開する場合**は、Google Apps Script の**プロジェクトをブラウザで開いたうえで**、メニューから **ウェブアプリとしてデプロイ**します。初めて開くと「権限の確認」などの案内が出ることが多いです。**操作の詳細は、この節の先頭のリンク**（とくに [webapp-deploy-for-beginners.md](docs/webapp-deploy-for-beginners.md)）を参照してください。
3. 次の **「初期設定画面」** で、フォームとスプレッドシートの URL を登録します。

### 2. 初期設定画面の開き方

| 状況 | どう開くか |
|------|------------|
| **初回**（管理者メールがまだ登録されていない） | Web アプリの **通常の URL** を開くだけでよい。アドレスに **`?page=setup` は不要**で、自動的に初期設定画面が表示される |
| **2 回目以降・設定の変更** | URL の末尾に **`?page=setup`** を付けて開く。**初回に設定を保存した Google アカウント**だけが開ける |

回答者に配るのは、初期設定が終わったあとに画面に表示される **回答用の URL**（`?page=setup` が付いていない通常のアドレス）です。

### 3. 初期設定画面に表示される内容

画面上の見出しと入力の意味は次のとおりです。

#### 初期設定の前に準備しておくもの

- **Googleフォーム**  
  質問項目や回答形式を含めて、あらかじめ完成させておいてください。

- **回答を確認するための Google スプレッドシート**  
  回答内容やログを確認するために使います。事前に作成しておいてください。

#### 初期設定

フォームとスプレッドシートの URL を貼り付けて **「設定を保存する」** を押すと、**回答者に送る用の URL** が表示されます。

- **URL をコピー**してメールやチャットで共有できます。
- **QRコード**は、その回答用 URL をブラウザ内だけで QR 化するオプションです（外部サービスへ URL を送りません）。
- **位置情報（GPS）**を回答と一緒に残す場合は、初期設定画面でのオン／オフはありません。**Google フォーム側に GPS 用の設問を 1 つ足す**だけで有効になります（手順は下記「GPS による現在地取得」を参照）。

#### 入力欄

- **GoogleフォームのURL（必須）**  
  使いたい Google フォームをブラウザで開き、アドレス欄の URL をそのまま貼り付けてください。

- **GoogleスプレッドシートのURL（必須）**  
  ログ確認に使う Google スプレッドシートを開き、その URL をそのまま貼り付けてください。

保存に成功すると、以降は同じ Web アプリの通常 URL で **回答画面**が開くようになります。

## GPS による現在地取得（任意）

授業出席など、**送信時点のおおよその位置を回答と同じ行で残したい**場合に使えます。Google フォーム単体では難しい処理を、**ラッパー（この Web アプリ）が送信直前にブラウザの位置情報 API で取得し、フォーム回答として保存**します。

### 使い方（GPS 機能をつけるには）

**前提:** この Web アプリのデプロイと、初期設定（フォーム・ログ用スプレッドシートの登録）が済んでいること。

1. **Google フォームを編集**し、次のとおり **1 問だけ** 追加する。  
   - **設問の種類:** **記述式（短文）**  
   - **タイトル（質問の見出し）:** **`GPS`** と **完全一致**（前後にスペースを入れない）。  
     - 以前から使っているフォームでは、タイトル **`__gps_latlng__`** でも同じ動作になります（互換用）。  
   - **必須にしない**（任意のままにする）。
2. **フォームを保存**する。
3. **回答者には、この Web アプリの回答用 URL（ラッパー URL）** だけを配る。  
   - ここから回答すると、ラッパーが位置を取り、上記の短文設問の列に **緯度,経度** を書き込みます。  
   - Google フォームの「プレビュー」や、フォーム本体の URL から直接送るだけでは、ラッパー経由にならないため **GPS は付きません**。
4. **（運用のコツ）** フォームの設問を並べ替えた・GPS 用の設問を追加した直後は、**管理者が一度だけ回答用ページを開き直す**と、内部の設問 ID の同期が確実です（通常は回答者が開いたタイミングでも同期されます）。

**オフにしたいとき:** フォームから GPS 用のその設問を削除するか、タイトルを `GPS` / `__gps_latlng__` 以外に変えると、ラッパーは位置取得を行いません。

### 特徴

- 回答者の画面に **GPS 用の入力欄は出ません**（ラッパーが描画から除外します）。
- **送信ボタンを押したあと**、位置の確認（最大約 8 秒）→ 送信、の流れになります。取得に失敗しても **回答の送信は続行**し、GPS 用の列は空になります。
- 値は **緯度と経度をカンマ区切りの 1 行**（例: `34.693725,135.502254`）で、フォームに連携した回答スプレッドシートの列に入ります。
- フォーム送信後にスプレッドシートへ **別途追記する方式は使いません**。

### 配布・見え方について

- 学生・回答者には **ラッパーの URL だけ** を配布するのがおすすめです。Google フォームの編集用 URL やフォーム単体の回答 URL を直接配ると、フォーム上に **「GPS」という設問タイトルがそのまま見える**ことがあります（中身はラッパーが埋めるため、直接フォームで答える必要はありませんが、紛らわしい場合はラッパー経由に統一してください）。

### 注意

- 位置情報は **ブラウザ・端末の許可と環境**に依存し、屋内などでは誤差が大きくなります。
- 厳密な証明には向かず、**出席などの補助情報**として扱う想定です。
- Web アプリは通常 HTTPS のため、位置情報 API と相性がよいです。

### うまくいかないとき

- フォームにタイトル **`GPS`**（または互換の `__gps_latlng__`）の短文設問があるか。
- 回答は **ラッパー URL** から行っているか。
- 列が空のときは、ブラウザで位置情報が拒否されたか、取得タイムアウトなどの可能性があります。
