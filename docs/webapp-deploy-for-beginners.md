# 編集画面から Web アプリを公開する（一般向け）

Google Apps Script（GAS）の**プロジェクトを、ブラウザ上で初めて開いたとき**は、コードの中身を理解しなくても、次のような流れになります。

1. **初めて開いた画面**で、Google から「このアプリを確認する」「権限を確認する」などの案内が出ることがあります。**組織のポリシーに従い**、必要なら IT 担当に相談しながら **承認** を進めてください。  
2. 承認が終わると **エディタ（コードが並んだ画面）** が使えるようになります。  
3. このツールをインターネット上の **URL で開けるようにする**には、メニューから **デプロイ（ウェブアプリとして公開）** を行います。

表示される文言は、Google の仕様変更で少しずつ変わることがあります。大まかな流れは次のとおりです。

---

## 手順の目安（画面の流れ）

1. 画面上部のメニューから **「デプロイ」** を開きます。  
2. **「新しいデプロイ」**（またはそれに近い名前）を選びます。  
3. **種類**の欄で、**歯車アイコン**や **「種類を選択」** から **「ウェブアプリ」**（Web アプリ）を選びます。  
4. 説明文を入力します（任意。後から識別しやすい名前でよいです）。  
5. **次の項目は組織のルールに合わせて選びます**（表示される選択肢の例）  
   - **次のユーザーとして実行** … 多くの場合は **「自分」** または **「アプリにアクセスしているユーザー」** のどちらかです。組織で指定があればそれに従ってください。  
   - **アクセスできるユーザー** … このリポジトリの想定では **組織内のユーザーのみ** などにすることが多いです。  
6. **「デプロイ」** を押します。  
7. 初回など、**アクセスを承認**する画面が出たら、案内に従って進めます。  
8. 完了すると **「ウェブアプリの URL」** や **「URL」** が表示されます。**そのリンクをコピー**して控えてください。これが、このツールを開くためのアドレスです。

**2 回目以降**にコードを直したあとで URL を更新したい場合は、再度 **「デプロイ」→「デプロイを管理」** などから、**新しいバージョン**でデプロイし直す操作が必要になることがあります。

---

## つまずきやすい点

- **権限の承認**で止まる … 個人用 Google アカウントではなく、**Workspace（会社・学校）のアカウント**でログインしているか、管理者に制限がないか確認してください。  
- **デプロイ**のメニューが見つからない … 画面右上の **「デプロイ」** ボタン、または **「拡張機能」** 周辺のメニューを確認してください（プロジェクトの種類によって表示が異なることがあります）。  
- **URL を開くとエラーになる** … デプロイ直後は数分かかることがあります。また **アクセスできるユーザー** の設定と、開いているブラウザのログインアカウントが一致しているか確認してください。
